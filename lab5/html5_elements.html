Implement each of the new elements you learned in lecture in your newly created document. Create a sound document that has the common elements (h1, h2, p, etc.), is semantic and well-structured, and contains comments.

  1. <abbr> Explains the definition of an abbreviation.

    ie: This is what WHATWG abbreviation means.

  2. <address> Represents the contact information for its enclosing section. If it is a child of the body element, then it applies to the document as a whole. And would appear usually in a footer for like contact email or links. Not to used for physical address.

    ie:

    Dave Raggett, Arnaud Le Hors, are contact persons for the W3C HTML Activity

  3. <b> Represents a span of text to be stylistically offset from the normal prose without conveying any extra importance. Examples are key words in a document abstract, product names in a review, or other spans of text whose typical typographic presentation is bold.

    ie:

  4. <i> Represents a span of text in an alternate voice or mood, or otherwise offset from the normal prose, such as a taxonomic designation, a technical term, an idiomatic phrase from another language, a thought, a ship name, or some other prose whose typical typographic presentation is italicized.

    ie: Last night I ate macarrones con queso. It was delicious.

  5. <code> The code element represents a fragment of computer code. This could be an XML element name, a filename, a computer program, or any other string that a computer would recognize. It is being used at the beginning of every li.

    ie: Every html document needs <doctype> at the top of the page.

  6. <del> The del element represents a removal from the document. The del elements should not cross implied paragraph boundaries

    ie: My favorite colour is orange green. Ahh! I love them both.

  7. <ins> The ins element represents an addition to the document. The ins elements should not cross implied paragraph boundaries.

    ie:

    My favourite colour is blue red, but I also like green and yellow.

  8. <details> The details element represents a disclosure widget from which the user can obtain additional information or controls. However, the details element is not appropriate for footnotes. Currently is only supported in Chrome.

    ie:

    Name of a good magazine

    Wired Magazine

  9. <summary> The summary element represents a summary, caption, or legend for the rest of the contents of the summary element's parent details element, if any.

    ie:

    More information

    Here is the source data that is discussed in the article ...

    ie: While they're essential reading material for our job, the W3C specifications are not exactly George R. R. Martin-level reading material.
